# AGENTS.md

## 项目交付规则

- 本仓库所有正式开发必须通过 Pull Request 交付，禁止直接向 `main` 提交功能代码。
- 每个议题从开发开始就应创建独立分支，并尽早打开 Draft PR。
- 开发周期内必须保持连续、可追踪的 commit 和 PR 更新记录。
- 禁止在最后一天一次性导入全部代码；此类提交不视为有效开发过程。
- 每个 PR 只处理一个明确范围，避免混入无关重构、格式化或临时文件。

## 分支与提交规范

- 功能分支命名建议使用 `feat/中文或英文主题`、`fix/中文或英文主题`、`docs/中文或英文主题`。
- commit message 使用 Conventional Commit 风格，并允许中文描述，例如：
  - `feat: 建立持续交付规范`
  - `feat: 添加项目说明文档`
  - `fix: 修复页面布局溢出`
  - `docs: 补充本地运行说明`
- PR 标题应使用同类风格，例如 `feat: 建立持续交付规范`。

## PR 内容要求

- PR 必须说明功能描述、实现思路、测试方式、风险与后续。
- PR 描述中的测试命令必须真实执行；无法执行时说明原因和残余风险。
- 合并前应确认目标分支可运行、无密钥、无本地日志、无构建产物和无无关文件。

## AI 助手执行约定

- 新对话进入本仓库后，应先阅读本文件并遵守上述规则。
- 除非用户明确要求，否则 AI 助手不得直接向 `main` 推送正式功能代码。
- AI 助手完成代码变更时，应提交到独立分支并创建 PR。
- 每次交付后，AI 助手应汇报分支名、commit、PR 链接和测试结果。
