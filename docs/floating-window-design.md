# Verba 悬浮窗视觉设计稿

> 基于 `product-requirements.md`、`requirement-review.md` 与 `执行mvp.md`：MVP 聚焦 Windows 桌面端，覆盖 YouTube / 网课 / 会议录播等英文音频场景，核心体验是“一键开始监听，悬浮显示实时双语字幕，并对修正结果给出可信反馈”。

## 设计目标

- **覆盖但不打扰**：窗口长期压在视频或会议画面上，必须低噪、半透明、可读。
- **中文优先**：中文译文是主视觉，英文原文作为辅助对照。
- **一眼知道状态**：开始、监听、重连、停止、修正都要有明确状态颜色。
- **突出竞赛创新点**：修正不是普通文本更新，要有短暂、克制的高亮反馈。

## 配色提取

来自用户参考图中的七牛云 / XEngineer 视觉：高饱和科技蓝、浅蓝信息底、金黄强调、白底黑字，以及当前 Verba 悬浮窗需要的深色玻璃遮罩。

| Token | 色值 | 用途 |
|---|---:|---|
| Brand Blue | `#1267FF` | 主按钮、品牌标识、主状态 |
| Deep Blue | `#0046B8` | 标题渐变、边框暗部、按压态 |
| Soft Blue | `#EAF3FF` | 亮色信息底、字幕修正淡底 |
| Text Blue | `#BFD8FF` | 深色悬浮窗上的中文译文 |
| Accent Yellow | `#FFD85A` | 修正高亮、关键词下划线、重要提示 |
| Success Green | `#12C76A` | WASAPI/系统音频捕获成功 |
| Danger Red | `#EF3B3A` | 停止、错误、断开风险 |
| Glass Black | `#101318` | 悬浮窗主体底色 |
| Ink White | `#F7FAFF` | 英文原文与主要文本 |
| Muted Gray | `#8A94A6` | 辅助说明、状态小字 |

## 悬浮窗结构

### 默认尺寸

- Windows Demo 默认窗口：`360 x 640`
- 主内容内边距：`20px`
- 字幕区：从顶部标题下方延伸至底部控制区上方
- 底部控制区高度：约 `116px`

### 布局

```text
┌─────────────────────────────┐
│ Verba                       │  品牌标题
│ AI 实时双语字幕助手          │  状态副标题
│ ── 蓝黄短线                 │
│                             │
│  "English original..."      │
│  中文翻译内容                │
│                             │
│  修正条：黄侧线 + 淡黄闪烁    │
│                             │
│ 已收到 5 条字幕              │
│                             │
│  系统音频按钮      服务按钮   │
└─────────────────────────────┘
```

## 关键状态

| 状态 | 视觉 |
|---|---|
| Idle | 深色玻璃底，中央提示“点击下方按钮开始监听”，主按钮为品牌蓝 |
| Listening | 顶部出现蓝黄短线，译文使用浅蓝，底部显示字幕计数 |
| WASAPI Capturing | 系统音频按钮为绿色，音量条使用绿到黄的渐变 |
| Reconnecting | 顶部副标题改为连接中，按钮保持停止红，字幕保留 |
| Corrected | 单条字幕左侧金黄竖线，背景淡黄闪烁，译文前显示修正图标 |
| Stopped | 字幕保留，可回看；主按钮回到品牌蓝 |

## 字幕样式

- 英文原文：`13px / 1.35`，白色 `90%` 透明度，中等字重。
- 中文译文：`16px / 1.45`，`#BFD8FF`，字重 `600`。
- 每条字幕间距：`10px`。
- 字幕条不做明显卡片，避免长期覆盖视频时太重。
- 修正旧译文保留一行删除线，颜色降低到 `#8A94A6` 透明态。

## 交互说明

- 底部左侧：系统音频捕获 / WASAPI 测试按钮。
- 底部右侧：服务连接 / 停止服务按钮。
- 用户上滑字幕列表后暂停自动滚底，右下角出现品牌蓝“回到底部”圆形按钮。
- 修正动效时长建议 `900-1200ms`，避免 500ms 过短导致用户错过，也避免整窗闪烁。
