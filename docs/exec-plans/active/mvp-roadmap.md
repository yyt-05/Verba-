## 第1步：最小主链路（Phase 1 Core Pipeline）

**目标**：用麦克风跑通「语音进入 → 中文字幕出现」的完整闭环。

**为什么先用麦克风？** WASAPI 的音频格式转换和编码需要额外调试。第一步先用 Flutter `record` 包从麦克风录音，把后端 ASR + 翻译 + SSE 整条链路跑通，回头再切到 WASAPI。

### 任务清单

| # | 任务 | 说明 |
|---|------|------|
| 1.1 | 安装 `record` 包 | `pubspec.yaml` 中加回 `record: ^7.0.0`（解决版本兼容问题） |
| 1.2 | 客户端麦克风录音 + 音频 chunk 上传 | 每攒够 2 秒 PCM 数据就 POST 一份到后端 |
| 1.3 | 后端 Whisper ASR 接入 | 真实调用 OpenAI Whisper API，英文音频 → 文本 |
| 1.4 | 后端 GPT-4o-mini 翻译接入 | ASR 英文文本 → 中文翻译 |
| 1.5 | SSE 推送双语字幕 | `subtitle.final` 事件携带 segmentId + original + translation |
| 1.6 | 客户端字幕列表渲染 | 收到 subtitle 事件 → 追加到列表 → 自动滚动 |

### 第1步的"做到什么程度算完成"

- [ ] 打开 App → 点击「开始监听」→ 对着麦克风说英文 → 3-5 秒内屏幕上出现英文字幕 + 中文翻译
- [ ] 持续说 5 分钟不中断、不崩溃、不乱序
- [ ] 停止后不再调用 API（验证 session stop 逻辑）
- [ ] `dart analyze` 零错误，`go build` 零错误

## 第2步：修正引擎（Phase 2 Correction）

**目标**：实现核心差异化能力——后验自动修正。

### 任务清单

| # | 任务 | 说明 |
|---|------|------|
| 2.1 | 修正引擎 LLM 调用 | 用 GPT-4o-mini，prompt 已在 `corrector.go` 中预留 |
| 2.2 | 滑动窗口触发逻辑 | 12 句窗口、每 3 句触发、回溯 6 句 —— 调参验证 |
| 2.3 | revision 版本控制 | 修正事件必须带 `revision` 字段，低版本修正丢弃 |
| 2.4 | 乐观锁 | 修正请求带 `window_hash`，窗口过期修正丢弃 |
| 2.5 | 客户端修正渲染 | 收到 `subtitle.corrected` → 按 segmentId 定位 → 更新译文 → 蓝色闪烁动画 |
| 2.6 | 修正降级开关 | 后端连续 3 次修正失败 → 暂停修正 30 秒 → 自动恢复。前端加开关让用户可关闭修正 |

### 第2步的"做到什么程度算完成"

- [ ] 对着麦克风说一段英文技术内容，故意包含术语（如 "attention mechanism"、"backpropagation"）
- [ ] 系统先给出初始翻译，之后收到 correction 事件，字幕原地更新为更准确的翻译
- [ ] 被修正的字幕条出现蓝色左竖条 + 闪烁动画 + 修正图标
- [ ] 修正不会导致重复字幕、不会导致列表滚动错位
- [ ] 修正降级开关可用——关闭后不再出现 correction 事件

## 第3步：异常与稳定性（Phase 3 Resilience）

**目标**：让 Demo 在真实场景下不容易翻车。

### 任务清单

| # | 任务 | 说明 |
|---|------|------|
| 3.1 | 网络断线自动重连 | SSE 断开 → 指数退避重连 → 字幕区显示「连接中...」 |
| 3.2 | 静音检测 + 提示 | 30 秒无语音输入 → 显示「未检测到语音」横幅 |
| 3.3 | ASR/翻译 API 重试 | 失败重试 3 次，还失败跳过当前句 |
| 3.4 | API Key 启动校验 | 启动时检查 Key 有效性，无效直接提示，不进主界面 |
| 3.5 | 权限拒绝引导 | 麦克风权限被拒 → 弹窗引导去系统设置 |
| 3.6 | 连续运行 1 小时 | 无崩溃、无内存泄漏、无字幕乱序 |

### 第3步的"做到什么程度算完成"

- [ ] 手动断网 → 看到「连接中...」→ 恢复网络 → 自动重连继续出字幕
- [ ] 静音 30 秒 → 看到提示 → 重新说话 → 提示消失
- [ ] 用无效 API Key 启动 → 直接显示错误页，不进主界面
- [ ] 连续运行 1 小时 → 内存使用稳定，字幕正常滚动

## 第4步：WASAPI 切换 + 体验打磨（Phase 4 Polish）

**目标**：从麦克风切换到系统音频，打磨悬浮窗体验。

### 任务清单

| # | 任务 | 说明 |
|---|------|------|
| 4.1 | WASAPI → 后端 音频流对接 | 替换麦克风录音，将 WASAPI 捕获的音频 chunk 上传到后端 |
| 4.2 | 音频格式转换 | WASAPI 输出 float32 → 编码为后端所需格式（PCM16 16kHz mono） |
| 4.3 | 字幕字号/透明度设置 | 三档字号 + 透明度滑块 |
| 4.4 | 悬浮窗拖拽 | 窗口可在屏幕上自由拖动 |
| 4.5 | 折叠/展开 | 空闲态小圆点 ↔ 激活态字幕窗口 |
| 4.6 | 回看暂停自动滚动 | 用户手动上滑 → 暂停自动滚底 → 出现「回到底部」按钮 |

### 第4步的"做到什么程度算完成"

- [ ] 打开 YouTube 视频（**无 CC 字幕的英文视频**）→ 悬浮窗实时显示中英双语字幕
- [ ] 窗口可拖动、可折叠、可调字号
- [ ] 手动上翻回看历史字幕 → 不会被打断 → 点「回到底部」继续实时滚动
