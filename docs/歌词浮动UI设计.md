# Verba 歌词式悬浮 UI 设计稿

> 目标：悬浮窗不要像工具面板，不要整块黑框。打开后先是一个很小的悬浮入口；开始监听后像桌面歌词一样，只显示中英双语字幕，依靠文字阴影、描边和轻量背景增强可读性。

## 设计关键词

- **透明悬浮**：默认没有黑色大框，不遮挡视频主体。
- **歌词式字幕**：内容居中或底部居中，像音乐播放器桌面歌词。
- **小入口**：打开 App 后先看到一个小圆形/胶囊悬浮按钮。
- **中文主视觉**：中文译文更大，英文原文更小。
- **轻控制**：控制按钮平时隐藏，鼠标悬停或点击才展开。
- **修正可感知**：修正时只高亮被修正的文字，不闪整个窗口。

## 核心配色

延续参考图中的七牛云 / XEngineer 气质，但在透明字幕场景里降低色块使用。

| Token | 色值 | 用途 |
|---|---:|---|
| Brand Blue | `#1267FF` | 开始按钮、品牌点、主进度线 |
| Lyric Blue | `#BFD8FF` | 中文译文 |
| Ink White | `#F7FAFF` | 英文原文、图标 |
| Accent Yellow | `#FFD85A` | 修正提示、关键词短线 |
| Danger Red | `#EF3B3A` | 停止按钮 |
| Shadow Black | `#05070A` | 文字投影，不作为面板底色 |
| Frost | `#FFFFFF` 12%-18% | 小控件毛玻璃底 |

## 整体信息架构

```text
打开 App
  ↓
小悬浮入口
  ↓ 点击开始
歌词式字幕悬浮
  ↓ 鼠标悬停 / 点击
轻控制条展开
  ↓ 停止
回到小悬浮入口，保留最近一句可选
```

## 状态一：未开始，小悬浮入口

### 视觉

```text
        ┌──────────────┐
        │  ▶  Verba    │
        └──────────────┘
```

- 尺寸：`112 x 42`
- 形态：胶囊按钮，不用大面板。
- 背景：白色 `14%` 透明毛玻璃，1px 浅蓝描边。
- 左侧图标：播放 / 麦克风图标，品牌蓝圆底。
- 文案：`Verba`，白色。
- 位置：默认屏幕底部居中上方 `120px`，也可拖动。

### 交互

- 点击胶囊：开始监听。
- 鼠标悬停：胶囊轻微放大，出现阴影。
- 右键或更多按钮：展开设置，MVP 可先不做。

## 状态二：监听中，歌词式字幕

### 常规展示

```text
               "The model uses attention to align tokens."
            该模型使用注意力机制来对齐词元。

                         ●  Listening
```

### 视觉规则

- 无外层黑框。
- 字幕整体宽度：屏幕宽度的 `64%-76%`，最大 `920px`。
- 英文原文：
  - 字号：`15px`
  - 颜色：`#F7FAFF`
  - 字重：`600`
  - 透明度：`88%`
- 中文译文：
  - 字号：`24px`
  - 颜色：`#BFD8FF`
  - 字重：`800`
  - 行高：`1.32`
- 文字保护：
  - 双层阴影：黑色强阴影 + 蓝色弱辉光。
  - 可选 1px 暗色描边，确保浅色视频上可读。
- 当前句最多显示 2 行中文，超出自动换行。
- 只显示最近 1 条主字幕；上一句以更小透明度向上淡出。

### 布局

```text
上一句淡出区（可选）
当前英文原文
当前中文译文
状态小点 / 延迟提示
```

## 状态三：鼠标悬停，轻控制条

### 展开形态

```text
            "The model uses attention to align tokens."
         该模型使用注意力机制来对齐词元。

       ┌────────────────────────────────────┐
       │  🎧 系统音频   ● 5条   A-  A+   ■  │
       └────────────────────────────────────┘
```

- 控制条不常驻，只在鼠标悬停字幕区域或点击状态点时出现。
- 控制条背景：白色 `12%` 透明毛玻璃，不用黑色实底。
- 圆角：`8px`
- 控件：
  - 系统音频状态
  - 字幕计数
  - 字号减小 / 增大
  - 停止按钮
- 停止按钮用红色小圆按钮，不要大红色块。

## 状态四：修正反馈

### 视觉

```text
            "Downward spiral."
         螺旋式下降。   ✦
         原译：向下螺旋。
```

- 被修正的中文译文短暂变为金黄 `#FFD85A`。
- 文字下方出现 2px 金黄短线，从左到右滑入。
- 右侧出现 `auto_fix_high` 修正图标，停留约 `1.2s`。
- 旧译文只在 1.5 秒内以小号灰字展示，然后淡出。
- 不闪整屏，不给整句加大色块。

## 状态五：连接中 / 异常

### 重连

```text
              正在恢复连接...
           最近字幕会保留在这里
```

- 字幕位置不变。
- 文案使用金黄，下面一条极细蓝色进度线循环。

### 音频源不可用

```text
              未检测到系统音频
           点击下方按钮重新检测
```

- 用简短文案，不弹大面板。
- 小控制条自动展开，突出“重新检测”按钮。

## 推荐尺寸

| 元素 | 桌面推荐 |
|---|---:|
| 小入口 | `112 x 42` |
| 字幕最大宽度 | `920px` |
| 字幕最小宽度 | `360px` |
| 控制条 | 自适应，约 `360 x 42` |
| 中文字号 | `22 / 24 / 28` 三档 |
| 英文字号 | 中文字号的 `62%-68%` |
| 默认位置 | 屏幕底部居中，距底 `110px` |

## Flutter 实现建议

### 窗口

- 窗口本身保持透明背景。
- 不再让整个 `HomePage` 使用 `glassBlack` 实底。
- 主体使用 `Stack`：
  - `LyricSubtitleLayer`
  - `MiniFloatingPill`
  - `HoverControlBar`
  - `StatusToast`

### 可读性

字幕文字统一加：

```text
TextShadow 1: #05070A 90%, blur 8, y 2
TextShadow 2: #1267FF 28%, blur 14, y 0
```

必要时增加 `StrokeText` 或使用 `Stack` 叠两层文本实现描边。

### 状态切换

- `idle / stopped`：只显示 `MiniFloatingPill`。
- `connecting`：显示歌词位置的连接文案 + 控制条。
- `listening`：显示歌词字幕。
- `hover`：显示控制条。
- `corrected`：触发金黄文字和短线动画。

## 与上一版黑框方案的差异

| 旧方案 | 新方案 |
|---|---|
| 360x640 深色工具窗 | 透明歌词式悬浮 |
| 长字幕列表常驻 | 当前句为主，上一句淡出 |
| 底部两个大圆按钮 | 小入口 + 悬停控制条 |
| 黑色容器保证可读 | 文字阴影/描边保证可读 |
| 更像调试工具 | 更像可长期覆盖视频的字幕产品 |

## 第一版落地优先级

1. 小悬浮入口。
2. 监听后显示无黑框双语歌词。
3. 鼠标悬停展示控制条。
4. 修正金黄高亮。
5. 字号三档与透明度设置。
