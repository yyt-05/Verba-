# Verba 实时同传：ASR+翻译+修正 API 选型对比表（适配你的2秒音频分片、后置纠错架构、低成本MVP）
> 项目需求约束：短分片(2s PCM)流式识别、短句实时翻译、支持上下文批量修正、按调用计费、适配Go后端批量调用，分 **ASR语音识别**、**LLM翻译+纠错** 两大块，Phase1云端API，后续可无缝切本地离线模型。

### ASR选型结论：
理由：你产品一大场景是手机外放收音（噪音大），Whisper脏音频识别能力＞国内厂商，出错样本刚好用来验证你的**后置LLM纠错核心功能**；入参格式统一，现有`audio/chunk.go`需补充multipart上传逻辑、无需大幅改编码。
**备用降级：Groq Whisper（降成本）、阿里云ASR（国内断网兜底）**

### LLM选型结论：
1. **一期固定：GPT-4o-mini**
现有`pipeline/translator.go、corrector.go`代码prompt全基于4o-mini调试，技术类英文纠错效果最优，能最大化凸显「上下文回溯修正」产品亮点，做演示、竞赛、面试项目最优；**ASR、翻译、修正都未真实接入，需要补完请求实现和错误处理**。
2. **二期迭代：替换DeepSeek V4 Flash**
项目正式商用后切DeepSeek，直接砍70%翻译成本，入参格式微调即可无缝替换。

## 三、组合落地搭配方案（分MVP演示版 / 商用正式版）
### 方案A：MVP演示、竞赛、开发调试（当前你优先落地）
- ASR：OpenAI Whisper API
- 翻译+纠错：GPT-4o-mini
- 优势：**需要补完请求实现和错误处理，完成对接后纠错效果拉满，演示修正动画效果最明显，面试/产品演示效果天花板**；
- 成本参考：连续1小时网课≈**2.66-3.31 元/小时**（修正原美元折算错误）。

### 方案B：低成本商用上线组合
- ASR主力：Groq Whisper Large v3 Turbo（低价提速，留意10s最低计费短音频损耗），兜底：阿里云英文ASR
- LLM主力：DeepSeek V4 Flash，兜底：通义千问mini
- 优势：综合成本下降60%+，国内网络稳定，面向普通用户包月无成本压力。

## 四、补充落地优化（适配你的后端架构）
1. **后端加API负载降级逻辑（补充进开发规范）**
在`pipeline`模块增加多服务商配置，GPT超时自动切DeepSeek，Whisper超时自动切Groq，环境变量配置服务商优先级，符合你「配置不硬编码」规范；
2. **token节流优化（降低计费）**
修正接口批量合并6条文本一次性入参，不要单句请求，大幅减少LLM调用次数（你现有滑动窗口设计天然适配批量请求）。

## 五、避坑提醒
国内厂商ASR/LLM不适合做主力：**技术网课大量AI/计算机专业英文，国产模型术语翻译错误率偏高，会导致你的修正引擎被无效错误填满，产品核心亮点无法体现**。
