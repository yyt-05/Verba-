# 一、AI 实时同传高频错译场景（正好对应你的「后置修正」核心价值，一共 6 大类，全是靠后文上下文才能改对）

## 1. 一词多义（最高发错误，技术网课重灾区）

英文单词多词性、多释义，**刚出场没上下文只能瞎选词义，后文出来才能修正**

示例：

`model`

- 前面单句：_The model is good._ → 初翻：**模特很不错**
- 后文：_used in transformer network_ → AI 修正 → **模型效果优异**

`bank`：河岸 / 银行；`agent`：代理 / 智能体；`fine`：精细 / 罚款；`token`：令牌 / 代币 / 词元

> 痛点：单句信息不足，必须靠后面句子回溯改译，是你修正引擎最核心落地场景。

## 2. 代词指代不明（it/this/that/they）

单独一句不知道代词指代前面哪个名词，初稿胡乱翻成「它、这个」，后文出现实体名词后修正。

> 例：
> 
> S1: It runs fast.【初稿：它运行很快】
> 
> S3: The transformer architecture runs fast.【修正：该 Transformer 架构运行效率很高】

## 3. 专业缩写、领域术语（技术视频刚需）

缩略词首次出现无全称，ASR 识别 + 翻译全部出错，后文展开全称再修正。

- ML：初稿「毫升」→ 后文 machine learning → 修正「机器学习」
- FFN：初稿「FFN 编号」→后文 Feed Forward Network→前馈神经网络

## 4. 英文长句拆分断句错误

英文复合长句被音频分片切成两段，前半段句子残缺，翻译语序混乱，后半句补齐全句后修正语序。

> 上半句：_If we apply the method which_ → 残缺乱译
> 
> 下半句：_is based on attention to NLP task_ → 拼接整句、理顺中文语序

## 5. ASR 语音识别本身出错（口音、连读、杂音、吞音）

麦克风外放收音 / 环境噪音 → Whisper 听岔单词，翻译跟着错；后文语义闭环，LLM 反向校正原文 + 译文。

- 原音：`parameter`（参数）
- ASR 误识别：`perimeter`（周长）→初翻：周长
- 后文：`model training hyper parameter` → 修正原文 + 译文：参数

> **手机外放收音场景这种错误巨多，你的修正就是产品救命功能**。

## 6. 倒装、从句、口语省略句式

英文口语倒装、省略主语，单句直译中文不通顺，补充上下文后润色本地化。

> Rare is this algorithm → 初稿：罕见是这个算法
> 
> 后文：used for lightweight inference → 修正：该算法在轻量化推理中并不常见

# 二、对应你的产品修正逻辑落地

1. 前端表现：**错误译文划线作废、黄色高亮替换新译文动画**；
2. 后端规则：滑动窗口缓存近 12 句，每新增 3 句触发 LLM 批量修正前面 6 句；
3. 产品卖点：市面上讯飞 / 沉浸式翻译**发现以上错误无法回溯修改，错译永久留在字幕**，是你的独家差异。
