# 产品需求

产品需求文档回答“做什么”和“为什么做”。它应该先定义目标用户、使用场景、MVP 边界、产品风险和验收标准，再进入实现细节。

## 基本原则

- MVP 聚焦 Windows 桌面实时双语字幕。
- 当前需求和未来想法必须分开写。
- TTS、移动端、登录、历史记录、商业化默认是独立需求，除非明确纳入当前范围。
- 扩大实现前，需求需要从前端、后端、设计、QA 视角评审。
